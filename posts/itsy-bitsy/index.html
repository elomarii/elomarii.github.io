ItsyBitsy

Put your ELK knowledge together and investigate an incident.

Posted Oct 26, 2024
By Ibrahim
1 min read

Checkout ItsyBitsy room on TryHackMe

I. Scenario

During normal SOC monitoring, Analyst John observed an alert on an IDS solution indicating a potential C2 communication from a user Browne from the HR department. A suspicious file was accessed containing a malicious pattern THM:{ ________ }. A week-long HTTP connection logs have been pulled to investigate. Due to limited resources, only the connection logs could be pulled out and are ingested into the connection_logs index in Kibana.

Our task in this room will be to examine the network connection logs of this user, find the link and the content of the file, and answer the questions.

II. Investigation

First, narrowing down the time periode where we’ll be investigating is very important. This way we can reduce noise and focus on what matters the most; Here, the target periode is march 2022:

1482 events (aka documents) are recorded for that period.

Bitsadmin is a command-line tool used to create, download or upload jobs, and to monitor their progress.

Looking at the user agents of recorded requests, we can notice that some requests were made with bitsadmin. HR people don’t use command-line tools so this must be suspicious.

We can filter the logs to show only request with bitsadmin as the user agent.

We end up with two requests, the source ip must be the one of the suspected user, i.e. 192.166.65.54.

From the same view, we can identify the filesharing website used to establish C2 connection, or to be more precise, to download a file that may be used to establish C2 connection, which is pastebin.com.

The full url to which the infected host connected is host + uri, i.e. pastebin.com/yTg0Ah6a.

We can access that url in a sandboxed environnement (to avoid the unknown) and see what kind of files the host did access.

The file secret.txt was downloaded, which contains the malicious pattern THM{SECRET__CODE}.
